# 自动记账 Skill

## 简介

当用户发送图片时，自动识别图片中是否包含记账信息（如支付截图、消费记录、账单等），提取关键信息后在一日记账APP中自动完成记录。

**让记账变得轻松，一张图片搞定！**

---

## 特性

- ✅ **自动识别** - 智能判断图片是否包含记账信息
- ✅ **多平台支持** - 支持微信、支付宝、京东、淘宝、美团等主流平台
- ✅ **智能分类** - 根据商家/用途自动推断消费分类
- ✅ **一键记账** - 自动在一日记账APP中完成记录
- ✅ **信息提取** - 提取金额、时间、商家、用途等完整信息

## 工作流程

```
┌─────────────────────────────────────────────────────────────┐
│                    用户发送图片                              │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  Step 1: 图像理解                                            │
│  - 调用 xiaoyi-image-understanding                          │
│  - 使用专用提示词识别记账信息                                 │
└─────────────────────────────────────────────────────────────┘
                              ↓
                    ┌─────────────────┐
                    │ 是否包含记账信息？│
                    └─────────────────┘
                     ↓ 是              ↓ 否
┌──────────────────────────┐    ┌──────────────────┐
│  Step 2: 信息提取与标准化  │    │  正常图片处理     │
│  - 金额标准化             │    │  不触发记账流程   │
│  - 分类智能推断           │    └──────────────────┘
│  - 时间格式化             │
│  - 描述优化               │
└──────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  Step 3: 构建记账指令                                        │
│  - 生成标准化的 GUI Agent query                              │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  Step 4: 执行记账                                            │
│  - 调用 xiaoyi_gui_agent 工具                                │
│  - 操作一日记账APP完成记录                                   │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│  Step 5: 结果反馈                                            │
│  - 记账成功：显示记录详情                                     │
│  - 记账失败：提示用户手动操作                                 │
└─────────────────────────────────────────────────────────────┘
```

## 用户偏好配置

可在 `memory/` 中记录用户的记账偏好：

```json
{
  "accounting_preferences": {
    "default_payment_method": "微信支付",
    "confirm_before_save": false,
    "custom_categories": {
      "我的分类": ["关键词1", "关键词2"]
    },
    "excluded_keywords": ["测试", "demo"],
    "large_amount_threshold": 1000
  }
}
```

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| default_payment_method | 默认支付方式 | 微信支付 |
| confirm_before_save | 保存前是否确认 | false |
| custom_categories | 自定义分类规则 | {} |
| excluded_keywords | 排除的关键词 | [] |
| large_amount_threshold | 大额阈值（需确认） | 1000 |

## 错误处理

| 错误类型 | 处理方式 | 用户提示 |
|----------|----------|----------|
| 无法识别金额 | 询问用户 | "无法识别金额，请告诉我具体金额" |
| 分类不明确 | 默认"其他" | "已记录为'其他'分类，可手动调整" |
| GUI操作失败 | 提供手动方案 | "自动记账失败，请手动记录：¥XX" |
| 图片模糊 | 请求重发 | "图片较模糊，请重新发送清晰图片" |
| APP未安装 | 提示安装 | "请先安装一日记账APP" |

## 最佳实践

1. **图片质量** - 确保图片清晰，关键信息可见
2. **完整截图** - 包含金额、商家、时间等完整信息
3. **及时记账** - 消费后立即发送截图，避免遗忘
4. **定期核对** - 定期在一日记账APP中核对记录
