# 自动记账 Skill

## 简介
当用户发送图片时，自动识别图片中是否包含记账信息（如支付截图、消费记录、账单等），提取关键信息后在一日记账APP中自动完成记录。

---

## 📝 Step 1: 图像理解提示词

### 主提示词（用于判断和提取）

```
请仔细分析这张图片，判断是否包含记账或账目相关信息。

【需要识别的内容】
1. 支付截图（微信支付、支付宝、银联等）
2. 消费小票/发票
3. 银行转账记录
4. 收款/付款凭证
5. 其他包含金额的交易记录

【购物平台订单截图】
特别注意识别以下平台的订单截图：
- 京东：订单详情页、支付成功页（识别"订单总额"、"实付款"）
- 淘宝/天猫：订单详情、付款成功页（识别"实付款"、"商品总价"）
- 拼多多：拼单成功、支付页面（识别"拼团价"、"实付"）
- 美团：外卖订单、到店付（识别"实付金额"、"订单金额"）
- 饿了么：外卖订单详情（识别"实付"、"合计"）
- 抖音电商：商品订单、直播购买（识别"实付金额"）
- 得物：商品订单（识别"支付金额"）
- 唯品会：订单确认页（识别"实付金额"）
- 盒马/叮咚买菜：生鲜订单（识别"实付金额"）

【如果包含记账信息，请提取以下内容】

1. 金额（amount）：
   - 提取数字金额，注意区分元/角/分
   - 如果有"-"或"支出"字样，标记为支出
   - 如果有"+"或"收入"字样，标记为收入

2. 交易类型（type）：
   - 支出：支付、消费、转账出去
   - 收入：收款、退款、转账进来

3. 商家/用途（merchant/description）：
   - 商家名称（如：星巴克、美团、滴滴）
   - 或用途描述（如：午餐、打车、购物）

4. 时间（time）：
   - 交易时间，格式化为 YYYY-MM-DD HH:mm
   - 如果没有时间，标记为 null

5. 支付方式（payment_method）：
   - 微信支付、支付宝、银行卡、现金等

【返回格式】
请严格按照以下JSON格式返回，不要添加任何其他内容：

{
  "is_accounting": true,
  "amount": 28.50,
  "type": "支出",
  "merchant": "星巴克",
  "description": "咖啡",
  "time": "2026-04-02 13:45",
  "payment_method": "微信支付",
  "confidence": 0.95
}

如果不包含记账信息，返回：
{
  "is_accounting": false,
  "reason": "图片内容描述..."
}
```

### 时间处理规则

1. **有明确时间**：直接使用，格式化为 `YYYY-MM-DD HH:mm`
2. **只有日期**：补充当前时间，如 `2026-04-02 00:00`
3. **无时间信息**：使用当前时间

### 描述优化

将商家名称和用途组合为清晰的描述：
- 有商家 + 有用途：`星巴克 - 咖啡`
- 只有商家：`星巴克`
- 只有用途：`咖啡消费`

```json
{
  "accounting_preferences": {
    "default_payment_method": "微信支付",
    "confirm_before_save": false,
    "custom_categories": {
      "我的分类": ["关键词1", "关键词2"]
    },
    "excluded_keywords": ["测试", "demo"],
    "large_amount_threshold": 1000
  }
```

### 偏好说明

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| default_payment_method | 默认支付方式 | 微信支付 |
| confirm_before_save | 保存前是否确认 | false |
| custom_categories | 自定义分类规则 | {} |
| excluded_keywords | 排除的关键词 | [] |
| large_amount_threshold | 大额阈值（需确认） | 1000 |
